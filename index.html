Restful API Notes

What is REST?

REST (Representational State Transfer) is an architectural style for building web services. It's a set of constraints used for creating efficient, reliable, and scalable web applications.

REST Principles

  • Client-Server Architecture: The client (frontend) and server (backend) are separate and independent.
  • Stateless: Each request from the client must contain all information needed to process it. The server does not store session state.
  • Cacheable: Responses can be explicitly defined as cacheable or non-cacheable to improve performance.
  • Uniform Interface (Key Feature): Uses standard HTTP methods (GET, POST, PUT, DELETE) to interact with resources.

HTTP Methods (CRUD Operations)

GET
Retrieves a resource (Read operation).
POST
Creates a new resource (Create operation).
PUT / PATCH
Updates an existing resource (Update operation).
DELETE
Removes a resource (Delete operation).
SWAGGER UI Notes

What is Swagger UI?

Swagger UI is an open-source tool that renders API documentation from an **OpenAPI Specification (OAS)** file in an interactive and user-friendly web interface.

Key Benefits

  • Interactive Testing: Allows developers and non-developers to send live requests to the API directly from the documentation page.
  • Standardized Format: Ensures documentation is always synchronized with the API logic when following the OAS standards.
  • Ease of Use: Provides a clear visual representation of API endpoints, parameters, and response structures.
CORS (Cross-Origin Resource Sharing) Notes

What is CORS?

CORS (Cross-Origin Resource Sharing) is a mechanism that allows a web browser to permit or deny web pages from one domain (origin) to request resources from another domain (cross-origin).

Relation to SOP

  • It is used to override the default browser security feature called the **Same-Origin Policy (SOP)**.
  • SOP blocks requests from one domain to another to prevent security risks like CSRF (Cross-Site Request Forgery).
  • CORS is necessary when your frontend (e.g., Vercel) is on a different domain than your backend API.

How it Works

The API server must send specific HTTP headers (like `Access-Control-Allow-Origin`) in its response, explicitly telling the browser which domains are allowed to access the resource.

GUI (Graphical User Interface) Notes

What is GUI?

A **GUI (Graphical User Interface)** is a system that allows users to interact with computers or software using visual representations like icons, windows, menus, and buttons, instead of typing text commands.

Key Elements

  • Visuals: Icons, windows, dialog boxes, and graphics.
  • Input: Buttons, checkboxes, text fields, and scroll bars.
  • User-Friendly: GUIs are the standard interface because they are intuitive and easy to navigate compared to the Command-Line Interface (CLI).
Laravel Framework Notes

What is Laravel?

Laravel is an open-source **PHP web application framework** known for its elegant, expressive syntax and use of the **Model-View-Controller (MVC)** architectural pattern.

Core Components

  • Eloquent ORM: An Object-Relational Mapper that allows you to interact with your database using clear PHP objects instead of writing raw SQL.
  • MVC: Separates application logic (Controller), data structure (Model), and presentation (View).
  • Artisan: A powerful command-line tool used to automate repetitive programming tasks, such as managing database migrations and creating boilerplate code.
POSTMAN Notes

What is Postman?

Postman is a collaborative platform for API development and testing. It allows developers to easily design, document, and debug various types of APIs (REST, SOAP, GraphQL).

Key Uses

  • API Testing: Used to send precise HTTP requests (GET, POST, etc.) and inspect the responses (status codes, body data, headers).
  • Collections: Organizes and groups requests for easy sharing and automated testing (test suites).
  • Environments: Enables the saving of variables (like URLs or API keys) to switch easily between development, staging, and production environments.
Bootstrap Notes

What is Bootstrap?

Bootstrap is the world's most popular front-end open-source toolkit for developing with HTML, CSS, and JavaScript. It provides a comprehensive collection of pre-designed components and templates.

Core Benefits & Features

  • **Mobile-First Approach:** Ensures sites look great on all devices, starting with small screens and scaling up.
  • **Responsive Grid System:** A powerful feature that allows for easy construction of flexible, responsive layouts using containers, rows, and columns (`col-md-6`, `col-lg-3`, etc.).
  • **Pre-built Components:** Speeds up development with ready-to-use components like Navbars, Buttons, Cards, and Modals.
  • **Utility Classes:** Provides helper classes for quick styling, such as managing spacing (`m-3`, `py-5`), colors (`text-bg-secondary`), and alignment.
