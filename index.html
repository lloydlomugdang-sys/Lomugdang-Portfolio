Restful API Notes

What is REST?

REST (Representational State Transfer) is an architectural style for building web services. It's not a protocol but a set of constraints used for creating efficient, reliable, and scalable web applications.

REST Principles

  • Client-Server Architecture: The client (frontend) and server (backend) are separate and independent.
  • Stateless: Each request from the client to the server must contain all the information needed to understand the request. The server cannot store client context between requests.
  • Cacheable: Responses can be defined as cacheable or non-cacheable to improve performance.
  • Uniform Interface (Key Feature): Uses standard HTTP methods (GET, POST, PUT, DELETE) to interact with resources.

HTTP Methods (CRUD Operations)

GET
Retrieves a resource (Read operation).
POST
Creates a new resource (Create operation).
PUT / PATCH
Updates an existing resource (Update operation).
DELETE
Removes a resource (Delete operation).
SWAGGER UI Notes

What is Swagger UI?

Swagger UI is an open-source tool that renders **API documentation** from an **OpenAPI Specification** (OAS, formerly known as Swagger Spec) file in a visually engaging and interactive way.

Key Benefits

  • Interactive Documentation: Allows developers and non-developers to interact with the API endpoints directly in the browser (e.g., testing GET and POST requests).
  • Standardized Format: Uses the OpenAPI Specification to ensure documentation is structured and machine-readable.
  • Easy Integration: Can be easily integrated into most web application frameworks (like Laravel, Node.js, etc.).

How it Works

  1. Developer creates an OpenAPI Specification (OAS) file (usually YAML or JSON) detailing the API endpoints, parameters, and response models.
  2. Swagger UI reads this file.
  3. Swagger UI generates a beautiful, interactive web page that lists the API resources and allows users to "Try it out."
